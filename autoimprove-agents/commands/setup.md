# autoimprove-agents: Setup

Interactive first-time setup for autoimprove-agents.

## Steps

1. Check if `~/.claude/autoimprove-agents/config.sh` already exists
   - If yes: ask "Config already exists. Reconfigure? (y/n)"
   - If no: proceed

2. Create data directory: `mkdir -p ~/.claude/autoimprove-agents/proposals`

3. Ask: "Do you want to enable NotebookLM integration? (requires notebooklm-py)"
   - If yes: check if `notebooklm-py` is installed (`python3 -c "import notebooklm"`)
     - If not installed: show `pip install notebooklm-py` and ask to retry or skip
     - If installed: ask for Google account email
   - If no: set `AUTOIMPROVE_NOTEBOOKLM_ENABLED=false`

4. Ask: "Auto-apply low-risk changes (archive tags, date updates)? (y/n, default: y)"

5. Ask: "Cross-project NotebookLM notebook name? (default: claude-cross-project-kb)"

6. Write config to `~/.claude/autoimprove-agents/config.sh`:
   ```bash
   # autoimprove-agents configuration (generated by /autoimprove-agents:setup)
   export AUTOIMPROVE_NOTEBOOKLM_ENABLED=<true|false>
   export AUTOIMPROVE_AUTO_APPLY_LOW_RISK=<true|false>
   export AUTOIMPROVE_CROSS_PROJECT_NOTEBOOK="<name>"
   export AUTOIMPROVE_GOOGLE_ACCOUNT="<email>"
   ```

7. Verify: source the config and confirm settings
8. Show: "Setup complete! Next steps:"
   - "Run `/autoimprove-agents:init` in any project to set up modular CLAUDE.md"
   - "Run `/autoimprove-agents:sync-kb` after sessions to capture knowledge"
